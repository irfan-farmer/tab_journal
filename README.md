# TabJournal

**Never lose a tab again — privately.** TabJournal periodically snapshots your
open tabs into a searchable, exportable history (a "journal") so you can find and
reopen any link you've had open. It runs entirely on your device.

> 🔒 **100% local.** No servers, no accounts, no analytics, no network requests.
> Your data never leaves your browser. URLs are stored without their query
> strings by default. See [PRIVACY.md](PRIVACY.md) — and verify it in the source.

## Why you can trust it

- **It can't phone home.** There is no network code anywhere in the source. It
  requests no host permissions, so it can't even read the contents of your pages —
  only tab titles and addresses.
- **Sensitive bits are stripped.** Query strings and fragments (where tokens and
  personal data live) are removed before storage, unless you opt in to full URLs.
- **You're in control.** Pause recording, exclude domains, export, or delete
  everything — all one click away.
- **It's small and readable.** A handful of short, commented files you can audit
  in a few minutes.

## What it stores

Timestamped snapshots of your open tabs — title, address (origin + path by
default), position, and pinned state — in the browser's local storage on this
device only. It records only `http`/`https` pages and skips any domains you
exclude. Full details in [PRIVACY.md](PRIVACY.md).

## Browser support

The JavaScript is browser-agnostic (`browser.js` selects the right API), so the
same code runs across browsers. `npm run build` produces a folder per target:

| Browser | Folder | Install (developer mode) |
|---|---|---|
| Chrome, Edge, Brave, Opera, Vivaldi | `dist/chrome` | `chrome://extensions` → enable Developer mode → **Load unpacked** |
| Firefox | `dist/firefox` | `about:debugging` → This Firefox → **Load Temporary Add-on** → pick `manifest.json` |
| Safari | from `dist/chrome` | `xcrun safari-web-extension-converter dist/chrome` (requires macOS + Xcode) |

Chromium browsers are the verified path; the Firefox and Safari targets share the
same code and should be smoke-tested before publishing to their stores.

## Files

- `manifest.json` — base (Chromium) MV3 manifest; the build derives other targets
- `browser.js` — cross-browser WebExtension API handle
- `background.js` — service worker: debounced, de-duplicated snapshots
- `privacy.js` — URL redaction and capture-eligibility rules (pure, auditable)
- `storage.js` — serialized write queue (`appendEntry`), `loadJournal`, `clearJournal`
- `config.js` — overridable settings with defaults
- `popup.html` / `popup.js` — search/reopen, pause, export, clear
- `options.html` / `options.js` — full settings + privacy controls
- `icons/` — generated by `scripts/make-icons.cjs`
- `scripts/build.cjs` — per-browser build into `dist/`
- `tests/parse_js.cjs` — acorn syntax check for all modules
- `tests/privacy.test.js` — `node:test` suite proving the URL-redaction rules

## Configuration

Defaults live in `config.js`; override them from the Options page (persisted in
local storage):

| Setting | Default | Meaning |
|---|---|---|
| `captureEnabled` | true | master on/off switch (popup "Active" toggle) |
| `captureEvents` | true | snapshot on tab create/remove/update |
| `snapshotMinutes` | 5 | periodic snapshot interval |
| `eventDebounceMs` | 2000 | collapse bursts of tab events into one snapshot |
| `maxEntries` | 1500 | journal cap before oldest entries are trimmed |
| `storeFullUrl` | false | keep full URLs (off = strip query/fragment) |
| `excludedDomains` | [] | hosts to never journal (subdomains included) |

## Scripts

- `npm test` — acorn parse-check of all modules + `node:test` redaction suite
- `npm run icons` — regenerate the PNG icons
- `npm run build` — produce per-browser folders in `dist/`
